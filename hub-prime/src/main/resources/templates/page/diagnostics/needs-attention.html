FHIR Submission Overview

Assumptions

  • The QE to Tech by Design File Count represents the total number of submissions received from each QE.
  • The Tech by Design Processed Count represents the number of submissions successfully processed by Tech by Design.
  • The Valid Bundles Count represents the number of processed submissions that passed validation.
  • The SHIN-NY Data Lake Count represents the number of valid bundles successfully submitted to the SHIN-NY Data Lake.
  • Each stage in the pipeline is expected to have the same count as the previous stage. Any mismatch between stages indicates a potential processing or validation issue.
  • When a mismatch occurs with the previous stage, the successful bundle count is displayed in red, and a drill-down arrow is enabled to view interaction records that were missed or not processed at that stage.
  • Clicking an interaction ID in the drill-down view opens a detailed popup displaying the full request and error information to assist with troubleshooting.
